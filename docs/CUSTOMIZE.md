# Customizing Lin OS

This document shows you how to customize Lin OS for your own use.

## Adding / removing packages

Edit `packages.x86_64`. Each line is a package name; lines starting with `#`
are comments. Group packages with section headers (also `#`-prefixed).

### Adding a package from the AUR

If the package isn't in `chaotic-aur`, you need to either:
1. Add it to `chaotic-aur` (not possible — that's someone else's repo), OR
2. Build it as a local package (like we do for `lin-branding`), OR
3. Add a one-time install in `airootfs/root/customize_airootfs.sh`.

Option 3 is the simplest for one-off packages:

```bash
# In airootfs/root/customize_airootfs.sh, add at the bottom:
echo "==> [lin] installing extra packages from AUR"
sudo -u lin yay -S --noconfirm some-aur-package
```

(You'd need to install `yay` first; bauh can also install AUR packages.)

## Changing the wallpaper

The wallpaper is generated procedurally by the `lin-branding` package's
PKGBUILD at build time, using ImageMagick. To change it:

1. Edit `packages/lin-branding/PKGBUILD`
2. Find the `convert -size 1920x1080 gradient:'#1a1d29'-'#6366f1' ...` line
3. Replace with your own `convert` invocation, OR
4. Replace with `cp "${srcdir}/my-wallpaper.png" "${pkgdir}/usr/share/backgrounds/lin/lin-default.png"`
   and add your wallpaper to the `source=()` array.

## Changing the GTK theme

The GTK theme is also generated by `lin-branding`'s PKGBUILD as a CSS file at
`/usr/share/themes/Lin-Dark/gtk-3.0/gtk.css`. Edit the CSS in the PKGBUILD to
tweak colors, padding, etc.

Reference: <https://docs.gtk.org/gtk3/css-properties.html>

## Changing the Hyprland config

The Hyprland config lives at `airootfs/etc/skel/.config/hypr/hyprland.conf`.
Key sections:

| Section | What it controls |
|---|---|
| `monitor=` | Monitor layout and scale |
| `general{}` | Gaps, borders, layout |
| `decoration{}` | Rounded corners, blur, shadows |
| `animations{}` | Animation curves and durations |
| `input{}` | Keyboard, touchpad, mouse |
| `bind = SUPER, ...` | Keybinds (see `keybinds.conf`) |
| `windowrulev2 = ...` | Per-app behavior (transparency, float, etc.) |

After changing, rebuild the ISO and test in a VM (QEMU, VirtualBox, VMware).

## Changing the waybar panel

The waybar config is at `airootfs/etc/skel/.config/waybar/{config,style.css}`.
- `config` (JSON) defines which modules appear and in what order.
- `style.css` defines the visual look.

To make the panel appear at the top instead of the bottom:
```jsonc
"position": "top",  // was "bottom"
```

To make it transparent instead of translucent:
```css
window#waybar {
    background-color: transparent;  /* was rgba(20, 22, 30, 0.65) */
}
```

## Changing the app launcher (wofi)

The wofi config is at `airootfs/etc/skel/.config/wofi/{config,style.css}`.

To change the width/height:
```
# In wofi/config
width=600    # was 480
lines=12     # was 8
```

## Adding your own desktop entry (dock icon)

Drop a `.desktop` file in `airootfs/etc/skel/.local/share/applications/`. It
will appear in wofi and any other app launcher.

Example — adding VS Code:
```ini
[Desktop Entry]
Type=Application
Name=VS Code
Comment=Visual Studio Code editor
Exec=code
Icon=visual-studio-code
Terminal=false
Categories=Development;IDE;
StartupNotify=true
```

## Replacing the app store

If you prefer `pamac-manager` as the default store instead of `bauh`:

1. Edit `airootfs/etc/skel/.local/share/applications/lin-store.desktop` and
   change `Exec=lin-store` to `Exec=pamac-manager`.
2. Or edit `/usr/local/bin/lin-store` (built by `lin-welcome` PKGBUILD) to
   swap the priority.

## Changing the SDDM login theme

The SDDM theme is shipped by `lin-branding` at `/usr/share/sddm/themes/lin/`.
The QML is in `main.qml`. Edit the colors / layout there.

To use a different SDDM theme entirely:
1. Add the theme package to `packages.x86_64`
2. Change `Current=lin` to `Current=<your-theme>` in `airootfs/etc/sddm.conf.d/lin.conf`

## Changing the Calamares installer branding

The installer branding is at `airootfs/etc/calamares/branding/lin/`:
- `branding.desc` — strings, colors, image paths
- `show.qml` — the slideshow shown during install
- `logo.png`, `welcome.png` — images

To change the slideshow text, edit `show.qml` and modify the `ListModel` at
the bottom.

## Adding your own systemd service

1. Drop the unit file at `airootfs/etc/systemd/system/your-service.service`
2. Enable it by adding to `customize_airootfs.sh`:
   ```bash
   systemctl enable your-service.service
   ```

## Building a custom-branded ISO

1. Fork this repo
2. Find/replace "Lin OS" → "YourBrand OS" everywhere (use `grep -r "Lin OS"`)
3. Replace the logo SVGs in `packages/lin-branding/PKGBUILD`
4. Change the wallpaper gradient colors
5. Change `profiledef.sh` `iso_name`, `iso_label`, `iso_publisher`
6. Change `airootfs/etc/os-release` `NAME`, `PRETTY_NAME`, `LOGO`
7. Push to your fork — GitHub Actions builds the new ISO

## Pinning to a specific kernel version

If you want to lock the kernel to a specific version (e.g. for stability):

1. Add `linux-lts` to `packages.x86_64` (instead of or in addition to `linux`)
2. In `customize_airootfs.sh`, set the default kernel in GRUB:
   ```bash
   mkdir -p /etc/default
   echo 'GRUB_DEFAULT="1>2"' > /etc/default/grub-lts-default
   ```
3. The Calamares bootloader module will pick this up.

## Removing gaming support (for a smaller ISO)

If you don't need Steam / Vulkan:

1. Remove these from `packages.x86_64`:
   - `vulkan-radeon`, `vulkan-intel`, `vulkan-nouveau`, `vulkan-swrast`
   - `lib32-mesa`, `lib32-vulkan-*`
   - `intel-media-driver`, `libva-*`
2. This saves ~300 MB on the ISO.

## Replacing Hyprland with KDE Plasma

If you want a more familiar desktop for Windows migrants (at the cost of
~400 MB more RAM at idle):

1. In `packages.x86_64`:
   - Remove: `hyprland`, `waybar`, `wofi`, `hyprpaper`, `hypridle`, `hyprlock`,
             `xdg-desktop-portal-hyprland`
   - Add: `plasma-desktop`, `plasma-nm`, `plasma-pa`, `sddm-theme-breeze`,
          `kwin`, `kded`, `xdg-desktop-portal-kde`
2. In `airootfs/etc/sddm.conf.d/lin.conf`, change `Session=hyprland` to
   `Session=plasmacompositor`.
3. Replace `airootfs/etc/skel/.config/hypr/` configs with KDE configs at
   `airootfs/etc/skel/.config/kdedefaults/`, `kwinrc`, `kscreenlockerrc`, etc.
4. Rebuild.
